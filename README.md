개요 : 라이브러리와 프레임워크를 이용해서 스마트팜 MES 구현 
역할 : 생산계획/작업지시/출하관리/주문관리/롯관리 페이지 풀스택 구현
- 활용기술 : html5, css3, javascript,java, Servlet, jsp, ajax, Apache Tomcat, spring, mybatis, REST API, sql, junit
⇨스프링, mybatis,lombok 등 다양한 라이브러리와 프레임워크를 이용해 프로젝트를 구현하면서 프레임워크와 라이브러리의 사용법을 익힐 수가 있었습니다.
팀원과 의견차로 갈등이 발생했을 때 선택 제시형으로 갈등을 최소화하는 전략을 취했습니다. 상대방이 원하는 의견만 채택되는 단점을 보완하기 위해 비대칭 장단점 부각 방식을 덧붙여 밸런스를 맞췄습니다.
이 프로젝트는 프로젝트 중에서 진척이 가장 느린 프로젝트였습니다. 원인 분석결과 제가 TDD를 하지 않고 당장 작동하는 코드만 짰더니 기능완성 이후에도 온갖 버그를 고치느라 시간을 쏟아야했습니다, 이미 완성된 기능을 봉인하지 않고 계속 수정하면서 다른 기능을 구현할 시간을 소모해버려 여타 기능을 구현하지 못했습니다.
해결책은1. TDD를 한다.
2. 완성된 기능은 봉인

이 두가지를 지키는 방식으로 이전보다는 기능 개발에 소요되는 시간이 줄어들었습니다. 앞으로는 TDD 외에 BDD도 신경쓰는 방향으로 발전하려고 합니다.
